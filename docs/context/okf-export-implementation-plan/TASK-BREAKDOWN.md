## PR: PR-2 — Storage protocol extension for arbitrary-path text IO

### Tasks
- **Extend `StorageProvider` protocol with `write_text` / `read_text` / `list_keys`** — Gate check: **FLAG** — touches a core provider abstraction; changing a `Protocol` forces coordinated updates across both implementations and risks silent breakage for any future provider
  - Relevant ADR/PRD sections: ADR-0019 §11; PRD-003 §Implementation Decision 11; Implementation Plan Q1-A
- **Implement `write_text` / `read_text` / `list_keys` on `LocalStorageProvider`** — Gate check: **FLAG** — state-management surface (filesystem tree under configurable root), path traversal risk if `root_dir` override is mishandled, must preserve existing source-upload behavior
  - Relevant ADR/PRD sections: Implementation Plan PR-2, Gotcha #1
- **Implement `write_text` / `read_text` / `list_keys` on `S3StorageProvider`** — Gate check: **FLAG** — S3 key-prefix logic, list pagination, and error handling; bugs here surface late (orphan files, partial exports)
  - Relevant ADR/PRD sections: ADR-0019 §11; ADR-0015; Implementation Plan PR-2, Gotcha #6
- **Add `tests/storage/test_export_io.py`** — Gate check: **PASS** — test scaffolding for the flagged implementations
  - Relevant ADR/PRD sections: Implementation Plan PR-2

### Flagged for Ticket Forge (pending user confirmation)
- `StorageProvider` protocol extension + local implementation — state-management surface, path traversal / isolation concerns, and a protocol change that binds all future providers
- `S3StorageProvider` text-IO implementation — S3 key-prefix and list behavior; failures tend to surface only during real S3-backed exports

### Fast execution track
- Add `tests/storage/test_export_io.py` (the tests themselves are not gated, but they validate the flagged implementations)

## PR: PR-4 — Manifest, per-page atomic write loop, orphan deletion, log writer

### Tasks
- **Implement `_Manifest` dataclass (load/save/removed_vs)** — Gate check: **FLAG** — state-management and data-integrity core; manifest corruption or mis-diffing causes duplicate `log.md` entries or missed orphan detection
  - Relevant ADR/PRD sections: ADR-0019 §7; PRD-003 §Implementation Decision 7
- **Implement `_LogWriter` (append_entry/flush)** — Gate check: **PASS** — append-only formatter; correctness depends on caller ordering in `_render_page`
  - Relevant ADR/PRD sections: ADR-0019 §7; Implementation Plan Q3
- **Implement `_render_page` with per-page atomic write ordering** — Gate check: **FLAG** — crash-safety invariant (file + log + manifest must stay in agreement); the hardest part of PR-4
  - Relevant ADR/PRD sections: ADR-0019 §10; PRD-003 §Implementation Decision 10
- **Implement orphan-deletion pass** — Gate check: **FLAG** — destructive filesystem/S3 operation driven by manifest diff; mistakes delete user-visible exported files
  - Relevant ADR/PRD sections: ADR-0019 §8; PRD-003 §Implementation Decision 8
- **Add tests for manifest, crash-safety, and orphan deletion** — Gate check: **PASS** — test scaffolding
  - Relevant ADR/PRD sections: PRD-003 §Testing Decisions

### Flagged for Ticket Forge (pending user confirmation)
- `_Manifest` + `_render_page` + orphan-deletion pass — these three define the export's crash-safety and consistency invariant; the production bug class is "stale/double/missing log entries and surprise file deletion"

### Fast execution track
- `_LogWriter` formatting
- Test scaffolding for manifest/crash-safety/orphan behavior
